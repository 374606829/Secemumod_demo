# 贡献指南

感谢你对 Secemumod Demo 的关注！欢迎通过 Issue 和 Pull Request 参与贡献。

## 如何贡献

### 提交 Issue

- **Bug 报告**：请描述复现步骤、环境信息、预期与实际行为
- **功能建议**：请说明使用场景与期望效果
- **文档问题**：指出错误或改进建议

### 提交 Pull Request

1. **Fork** 本仓库到你的账号
2. **创建分支**：`git checkout -b feature/your-feature` 或 `fix/your-fix`
3. **修改代码**：遵循现有风格，保持简洁
4. **提交**：`git commit -m "feat: 简短描述"`
5. **推送**：`git push origin feature/your-feature`
6. 在 GitHub 上发起 **Pull Request**，描述改动内容

### 提交信息规范

- `feat:` 新功能
- `fix:` 修复
- `docs:` 文档
- `style:` 格式（不影响逻辑）
- `refactor:` 重构

## 代码规范

- **Python**：遵循 PEP 8，保持可读性
- **Java**：遵循 Spring Boot 常见约定
- **提交前**：确保本地可运行，无新增 Lint 错误

## 联系方式

如有疑问，可通过 GitHub Issue 讨论。
